链表实战#

题目描述#

有 n 辆自行车依次来到停车棚，除了第一辆自行车外，每辆自行车都会恰好停放在已经在停车棚里的某辆自行车的左边或右边。(e.g.停车棚里已经有 3 辆自行车，从左到右编号为:3,5,1。现在编号为 2 的第 4 辆自行车要停在 5 号自行车的左边，所以现在停车棚里的自行车编号是：3，2，5，1）。给定n辆自行车的停放情况，按顺序输出最后停车棚里的自行车编号。n≤100000。

输入描述

第一行一个整数 n。 第二行一个整数x。表示第一辆自行车的编号。 以下 n−1 行，每行
3 个整数 x,y,z。 z=0 时，表示编号为 x 的自行车恰停放在编号为 y 的自行车的左边。 z=1 时，表示编号为 x 的自行车恰停放在编号为 y 的自行车的右边。

输出描述

从左到右输出停车棚里的自行车编号

输入样例

        4
    3
  1 3 1
  2 1 0
  5 2 1

输出样例

3 2 5 1

思路#

静态链表实现思路#

①：先初始化下标为0,1的链表结点。
如图

②：接下来我们从下标为2的链表结点开始，先将第一辆自行车插入结点0和结点1之间。之后保持结点0，1不动，结点0，1仅仅用于判别左右方向边界。

③：由于链表在插入的时候需要一个一个遍历过去，它的时间复杂度是O(n)，n题目说是≤100000 。当n过大时，容易引起程序运行超时。所以我们引进一个located[N]定位数组，用于定位每个自行车的位置。这样我们就可以根据自行车的名字直接定位到其结点位置进行插入操作，无需进行遍历查找。
④：结合以上三点，我们就可以实现对本题的代码实现

创建链表

struct Node
{
  int p;//存储上一个结点
  int N;//存储下一个结点
  int data;//存储自行车的编号
}Node[N];

创建一个locate数组用于存储自行车摆放的结点位置，便于快速定位

int locate[N];

编写一个初始化函数，设置链表结点0，1作为左右方向的边界

void init()
{ Node[0].N=1;
   Node[1].p=0;
   now=2;

void insert(int k,int a)//变量k是提供位置信息，例如x要插入到y的左边，那这个k就代表了y的位置信息。而变量a则提供新插入的自行车的编号
{ Node[now].data=a;//将新自行车的编号插入新结点的数据部分
 locate[a]=now;//将新的自行车结点的下标存储在locate数组，便于快速查找
 Node[now].N=Node[k].N;//将新结点的N（next）指向新结点右边的结点
 Node[Node[k].N].p=now;//将刚刚新节点的N指向的右边结点重新指回新结点，以构成双向链表，方便结点左右方向的任意插入
 Node[k].N=now;//将属于新节点左边的结点指向新结点
 Node[now].p=k;//新节点的P（preid）指向其左结点。
 now++;//结点位置+1，便于下一个新结点的创建。
}//这个代码需要结合思路部分的第二张图片去理解更佳

对于左右插入顺序的执行，在插入函数中最后的实现其实是偏右边的，比如k=0，那我们新的结点插入的位置是在0结点的右边的。右边插入我们解决了，接下来我们按照相对性的普遍规律，我们输入k=-1结点(这只是假设，假设0结点前面有一个-1结点)，那结点就会插入到-1结点的右边，这不就相当于在结点0的左边嘛。这样我们左边插入和右边插入的问题就迎刃而解了

源代码#

#include <iostream>
using namespace std;
const int N=120000;
int now;
struct Node
{
  int p;
  int N;
  int data;
}Node[N];
int locate[N];
void init()
{ Node[0].N=1;
   Node[1].p=0;
   now=2;
}

void insert(int k,int a)
{ Node[now].data=a;
 locate[a]=now;
 Node[now].N=Node[k].N;
 Node[Node[k].N].p=now;
 Node[k].N=now;
 Node[now].p=k;
 now++;
}



int main()
{
  int n,x,y,z;
  cin>>n;
  int a;
  cin>>a;
  init();
  insert(0,a);//将第一辆自行车插入数据
  n--;//第一辆插入了之后总数量要减1
  while(n--)
   {
      cin>>x>>y>>z;
      if(z==0)
      {
        insert(Node[locate[y]].p,x);
      }

    else
       {
           insert(locate[y],x);

       }

   }

   for(int i=Node[0].N;i!=1;i=Node[i].N)
     {
       cout<<Node[i].data<<" ";//将结点的数据依次输出
     }

  return 0;
}

C++STL 链表容器实现源代码#

#include <bits/stdc++.h>
using namespace std;
int main()
{ int n,x,y,z,a;
    list<int>::iterator locate[200000];//用于定位自行车对应的结点位置
    list<int> Node;
    list<int>::iterator temp;//迭代器，只访问物体的外部信息，而不访问内部信息，可以看作指针，指针是迭代器的一种
    cin>>n>>a;//a是第一类自行车的编号，n是总共的自行车数量
    Node.push_back(a);//将第一辆自行车的数据存入链表结点
    locate[a]=Node.begin();//将第一辆自行车的结点的位置信息存到locate 迭代器中，因为list容器访问的时候只能通过自增去一个一个查询，而不可以直接通过下标去访问。那样太慢了，我们直接通过存自行车编号对应的迭代器，可以直接定位到具体的结点位置，提高运行效率
    for(int i=1;i<=n-1;i++)
    { cin>>x>>y>>z;

        if(z==0)
        {  Node.insert(locate[y],x);//将编号x的自行车插到y编号自行车的左边
            temp=locate[y];   //存储y编号自行车的位置
            locate[x]=--temp; //temp位置是对应y自行车的，现在x编号的自行车已经在y自行车的前面了，所以只要将temp自减1，就是x对应的位置信息了
        }
        else
        { Node.insert(++locate[y],x);//将编号x的自行车插到y编号自行车的右边
            temp=locate[y];
            locate[x]=--temp;
        }
    }
    temp=Node.begin();
  for(temp;temp!=Node.end();++temp)//通过迭代器遍历输出结果
  {
      cout<<*temp<<" ";
  }

    return 0;
}

总结#

通过一locate数组/list迭代器 来存储位置信息，通过空间换时间可以用来解决链表遍历过慢的问题。本题核心的点在于利用双向链表，通过物体相对的位置关系去实现了左边插入和右边插入。
